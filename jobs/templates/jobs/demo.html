Foreman

An event-driven job pipeline you can watch: submit a property-CSV import and see it move through a transactional outbox, an idempotent worker, retries with backoff, a dead-letter queue, and operator redrive — live over a WebSocket, with the queue's own metrics ticking beside it.

Run a scenario

Happy path: outbox → relay → worker → SUCCEEDED, then a downloadable CSV report.

Fails, then retries with backoff and recovers automatically — watch attempt climb.

Exhausts its retries into DEAD_LETTER. Then redrive it — the operator recovery action.

Poison input fails fast (FAILED) with no wasted retries.
